# 태그 사전 (controlled vocabulary)

`facilities.tags text[]` 에 들어가는 **정해진 태그 목록.** 순수 태그 모델(카테고리 없음) — 목욕탕·찜질방·온천도 태그.

## 규칙
- **이 문서에 등록된 태그만 사용.** 프리폼 자유입력 금지("노천"/"노천탕"/"야외탕" 분열 방지).
- **새 태그 추가 = 이 문서에 먼저 등록** 후 코드/UI 반영.
- **대표**(⭐) = UI 홈에서 간판(둘러보기 진입)으로 노출하는 태그. 데이터는 평등, 노출만 승격.
- **출처**: `자동`=공공데이터 import 시 자동 부여 · `UGC`=유저 제보/방문기록으로 채움.

## 유형 (무엇인가)
| 태그 | 대표 | 출처 | 비고 |
|---|---|---|---|
| 목욕탕 | ⭐ | 자동 | 위생업태명 공동탕업·목욕장업 기타 |
| 찜질방 | ⭐ | 자동 | 찜질시설·공동탕업+찜질 |
| 한증막 | ⭐ | 자동 | 한증막업 |
| 온천 | ⭐ | 자동 | 별도 온천 데이터 |
| 스파 | | UGC | 호텔스파·프리미엄 |
| 핀란드사우나 | | UGC | 로우류 중심 신형(트렌드) |

## 자동 태깅 규칙 (import 시)
- 위생업태명 → 유형 태그: `공동탕업/목욕장업 기타→목욕탕` · `찜질시설/공동탕업+찜질→찜질방` · `한증막업→한증막`
- `발한실여부 = Y` → `건식사우나`
- 온천 데이터(별도) → `온천` + `온천수` (목욕업과 이름·주소 겹치면 병합)

## specs (태그 아닌 값) — 참고
태그는 예/아니오, **값은 `facilities.specs jsonb`**:
- `sauna_temp`(사우나 온도), `cold_temp`(냉탕 온도), `price_adult`(성인요금), `hours`(영업시간), `parking_note` 등
- 예: `{"sauna_temp": 90, "price_adult": 9000, "hours": "05:00-24:00"}`

## 확장 시 체크
- 자유태그 도입은 나중에(별도 `free_tags` 검토). MVP는 이 사전만.
- 태그가 너무 자주 필터되면 전용 컬럼/인덱스 승격 고려.
