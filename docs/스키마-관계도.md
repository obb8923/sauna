# 스키마 관계도 (ERD)

전체 스키마·정책은 [스키마.md](스키마.md). 아래는 관계 요약(Mermaid).

```mermaid
erDiagram
    profiles ||--o{ posts        : "작성"
    profiles ||--o{ reviews      : "평가"
    profiles ||--o{ wishes       : "찜"
    profiles ||--o{ post_likes   : "좋아요"
    profiles ||--o{ post_reports : "신고"
    profiles ||--o{ user_blocks  : "차단-가해"
    profiles ||--o{ user_blocks  : "차단-피해"

    facilities |o--o{ posts   : "방문기록(선택)"
    facilities ||--o{ reviews : "평가대상"
    facilities ||--o{ wishes  : "찜대상"

    posts ||--o{ post_likes   : ""
    posts ||--o{ post_reports : ""

    profiles {
        uuid id PK "auth.users 연동"
        text nickname
        text avatar_url
    }
    facilities {
        bigint id PK
        text name
        text tags "배열, 태그사전"
        text sido
        text sigungu
        float lat
        float lon
        geography geog "반경검색"
        jsonb specs "온도·요금·영업시간"
    }
    posts {
        bigint id PK
        uuid user_id FK
        bigint facility_id FK "NULL이면 자유글"
        text topic "자동파생: 방문기록/자유"
        text body
        text photos "배열, 경로"
        int like_count "트리거 동기화"
        int report_count "5건 자동 blocked"
        text status "visible/blocked/hidden"
    }
    reviews {
        bigint id PK
        uuid user_id FK
        bigint facility_id FK
        int rating "1~5"
        text body
    }
    wishes {
        uuid user_id PK_FK
        bigint facility_id PK_FK
    }
    post_likes {
        uuid user_id PK_FK
        bigint post_id PK_FK
    }
    post_reports {
        bigint id PK
        uuid reporter_id FK
        bigint post_id FK
        text reason
    }
    user_blocks {
        uuid blocker_id PK_FK
        uuid blocked_id PK_FK
    }
```

## 읽는 법
- `||--o{` = 1 : 다 (필수). `|o--o{` = 0또는1 : 다 (선택 — posts.facility_id는 NULL 가능).
- **profiles·facilities = 뼈대**, 나머지는 둘을 가리키는 연결.
- **posts**: facility 있으면 방문기록, 없으면 자유(topic 자동). **reviews**: 별점, 시설 필수, 1인 1시설(`unique(user_id,facility_id)`).
- **복합 PK**(PK_FK): wishes·post_likes·user_blocks는 두 FK 조합이 곧 기본키 → 중복 방지(같은 찜/좋아요/차단 1번).
- **트리거**: 로그인→profiles 생성, 좋아요→like_count, 신고→report_count·자동차단, lat/lon→geog.
