# 네비게이션 구조

앱(본체)과 웹(SEO 퍼널)은 성격이 달라 따로 설계. 데이터는 순수 태그 모델([태그-사전.md](태그-사전.md)), 시설평/방문일지 분리, 페이크도어 티저 반영.

## 앱 (Expo Router) — 탭바 3개
```
[ 탐색 ]        [ 피드 ]         [ 더보기 ]
 지도/리스트    방문일지 타임라인   마이+설정+약관+기타
        └─ 기록하기 FAB(+) 떠있음 ─┘
```

### ① 탐색 (홈, 첫 화면)
- 상단 공통: 검색바 + **대표 태그(⭐) 칩**(온천·찜질방·사우나·불가마·24시간) + 필터 시트(태그 그룹·지역)
- **[지도] / [리스트] 토글**
  - **지도**: 핀 + 내 주변(`nearby_facilities` RPC) + 태그 필터
  - **리스트**: **지역(시도) 선택**(서울·부산·경기…) → 그 지역 시설 리스트. `where sido = ?` + 태그 필터 + 정렬(가까운순/이름순)

### ② 피드 (방문일지 타임라인)
- 전체 유저 방문일지(visits) 최신순. 사진·시설명·N회차 (커뮤니티 엔진 노출)
- 팔로우 피드는 나중 — MVP는 전체

### ③ 더보기 (마이 흡수 + 캐치올)
- **내 활동(상단)**: 프로필 · 내 방문기록 · 찜 목록
- **설정**: 알림 · 계정 · 로그아웃 · 회원탈퇴(스토어 필수)
- **정책/약관**: 이용약관 · 개인정보처리방침 · 위치기반서비스 약관(위치 사용) · 오픈소스 라이선스
- **기타**: 공지사항 · 문의/피드백 · 앱 버전
- **티저(페이크 도어)**: 매거진 · 굿즈 (탭카운트 + 알림받기)

### 기록하기 FAB
- 탐색/피드에 "+" 상시 노출 → 시설 선택 → 방문기록. (핵심 액션이라 손 닿게. 시설상세에서도 진입 가능. 불필요하면 제거 쉬움)

### 스택 화면 (탭 위 push)
- **시설 상세** `facility/[id]`
  - 헤더: 이름 · 태그 칩 · 거리 · 전화 · 찜 버튼
  - 정보: specs(온도·요금·영업시간) · 지도 위치
  - **하위 탭: [시설평] | [방문일지]** ← 사우나이키타이가 못 한 분리
  - CTA: "방문기록 남기기" · "시설평 쓰기"
- **방문기록 작성** `visit/new` — 날짜·메모·사진 (3탭 내 마찰 최소)
- **시설평 작성** `review/new` — 별점 + 본문
- **검색/필터 결과** `search` — 태그(`tags @>`)·지역 필터 리스트
- **로그인/온보딩** `login` — Google·Apple 로그인(Supabase Auth)
- **약관/정책 상세** `legal/[doc]`

### 파일구조 (Expo Router)
```
src/app/
  _layout.tsx                (auth gate + stack)
  (tabs)/_layout.tsx         (탭바 3개 + FAB)
  (tabs)/index.tsx           탐색 (지도/리스트 토글)
  (tabs)/feed.tsx            피드
  (tabs)/more.tsx            더보기
  facility/[id].tsx          시설상세 (시설평|방문일지)
  visit/new.tsx
  review/new.tsx
  search.tsx
  login.tsx
  legal/[doc].tsx
```

## 웹 (Next.js) — SEO 퍼널, 읽기 전용·얇게
- `/` 홈: 지역·대표태그 진입 + 검색 + 지도
- `/f/[id]` **시설 상세 (SSG/ISR)** ← SEO 핵심. 정보 + 시설평 미리보기 + **앱 다운로드 CTA**
- `/지역/[sido]` 지역별 목록 (SSG) ← 롱테일 SEO (앱 리스트뷰와 대응)
- `/태그/[tag]` 태그별 목록 (SSG) ← 롱테일 SEO
- 정책 페이지(약관·개인정보) — 앱 legal과 공유
- **쓰기 기능 없음.** 모든 페이지에 앱 CTA(출시 전이면 "베타 알림받기").

## 요약
- 앱 탭 3개: 탐색(지도/지역리스트) · 피드(방문일지) · 더보기(마이+약관+티저)
- 핵심 차별점은 **시설상세의 시설평/방문일지 분리** + **기록 마찰 최소**
- 웹은 시설·지역·태그 SSG 페이지로 SEO 유입 → 앱 유도
