# 공공데이터 품질 & 시드 (bathhouses)

## 원본
- **전국목욕업소표준데이터** (LOCALDATA, `생활_목욕장업.csv`, CP949, 17,757행, 수시 갱신)
- 37컬럼. 영업상태·업태(위생업태명)·좌표(EPSG:5174 TM)·지번/도로명주소·발한실여부·욕실수 등
- 온천은 별도(행안부 전국온천이용업소현황 505행, 주소·전화만 / 전국온천현황은 PDF 통계라 정형 아님)

## 프로파일 (영업중 5,652 기준)
- 폐업 12,105 / **영업중 5,652** (폐업 68% → 필터 필수)
- 업태: 목욕탕/사우나 3,941 · 찜질방 1,141 · 한증막 132 · 기타 438
- **발한실(사우나설비) 보유 2,536**
- 지번주소 결측 0.2% · 좌표 결측 12% · 좌표 OR 지번 99.9% · 중복 2건 → **A급**

## ETL (→ `bathhouses.json` 시드)
`영업중 필터 + 업태 분류 + 발한실여부 + TM(EPSG:5174)→WGS84 변환`
- pyproj 설치 불가 환경 → TM 역투영 + Bursa-Wolf datum shift 순수 구현
- 결과: 5,652곳, **위경도 확보 4,940(87%)**, 좌표없음 712(지번 지오코딩 대기), 변환/범위이탈 0
- 정확도: 종로 내수동167 → 37.5728,126.9733(광화문) 일치 · 범위 33.22~38.51N, 124.71~130.91E 정상
- 레코드: name · category · sauna(bool) · address · phone · lat · lon

## 남은 처리
1. 좌표없음 712곳 → 지번주소 지오코딩(카카오 로컬 API)
2. 주소 → sido/sigungu 파싱(지역 필터용)
3. 온천 505곳 별도 붙이기(좌표 지오코딩 + 수질 수작업)
4. 시설 상세(노천·온도·불가마)는 공공데이터에 없음 → UGC(방문기록·제보)로 채움
